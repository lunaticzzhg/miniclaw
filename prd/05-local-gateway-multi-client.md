# P5 本地 Gateway 与多端连接

## 目标
让 MiniClaw 从单机 App 变成一个可被多个客户端访问的个人助手中心。

## 用户可获得的价值
- 手机不再只是界面，而是个人助手的控制中心之一
- 用户可以从多个端访问同一个助手
- 对话、状态、设备连接在一个统一视图中管理

## 交互设计
- 新增“连接中心”页面
- 页面展示 Gateway 状态、连接设备数、在线客户端列表
- 提供连接码或配对流程
- 用户可以查看某个客户端的在线状态与最近活动
- 用户可以手动断开某个客户端连接

## 本期范围
- 本地 Gateway 状态展示
- 多客户端连接管理
- 配对与连接流程
- 在线状态展示
- 手动断开连接

## 非目标
- 外部社交通道接入
- 自动化规则
- 插件市场

## 验收标准
- 用户可看到 Gateway 当前运行状态
- 至少支持 2 个客户端接入同一助手实例
- 客户端连接后，可共享同一会话体系
- 用户可主动断开某个客户端
- 连接异常时，有明确的错误反馈

## 测试方式
1. 启动后进入连接中心，验证状态可见
2. 使用第二客户端接入，验证配对流程完成
3. 在不同客户端查看同一会话，验证内容一致
4. 断开某个客户端，验证状态更新及时
5. 模拟网络中断，验证错误提示和恢复反馈
