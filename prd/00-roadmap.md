# MiniClaw 产品分期路线图

## 产品目标
MiniClaw 最终目标是完整复刻 OpenClaw 的核心能力，成为一个以移动端为主入口的个人 AI Assistant App。产品需要同时具备聊天、Agent 执行、工具调用、设备能力、多端连接、多渠道接入、自动化和插件扩展能力。

## 分期原则
- 每一期都必须可独立演示
- 每一期都必须有明确的用户价值
- 每一期都必须有可执行的验收测试
- 每一期先关注用户体验和产品闭环，不在 PRD 中讨论实现细节

## 分期列表
1. P1 会话与聊天基础
2. P1_1 真实模型链路与模型可插拔
3. P2 Agent 工具调用闭环
4. P3 多会话与记忆管理
5. P4 语音与设备能力
6. P5 本地 Gateway 与多端连接
7. P6 外部消息渠道接入
8. P7 自动化与主动任务
9. P8 Skills / Plugins 扩展体系
10. P9 控制台、可视化与产品打磨

## 完整复刻范围
最终版本应覆盖以下产品能力：
- 单用户个人 AI 助手
- 多会话与会话隔离
- 工具调用与结果回填
- 长短期记忆
- Android 设备能力接入
- 语音输入与播报
- 本地 Gateway 控制平面
- 多客户端连接
- 多消息渠道接入
- 定时任务与主动提醒
- Skills / Plugins 扩展
- 控制台与调试能力

## 完成定义
当 P1、P1_1、P2-P9 全部完成后，MiniClaw 应达到“独立作为个人 Agent 产品使用”的水平，并能对 OpenClaw 的核心使用路径完成产品层面的复刻。
