# P2 Agent 工具调用闭环

## 目标
让 AI 不只是回答问题，还能调用工具完成任务，形成最基本的 Agent 能力。

## 用户可获得的价值
- 用户可以让 AI 查询信息、记录内容、执行简单动作
- 用户能看到 AI 是否调用了工具，以及工具返回了什么结果
- 用户能区分普通聊天和任务执行型对话

## 交互设计
- 聊天页中，工具调用以独立卡片展示
- 工具卡片显示工具名、执行状态、结果摘要
- 对高风险工具，执行前弹出确认框
- 工具执行失败时，显示失败原因和重新执行入口
- AI 最终回复应基于工具结果给出整合答案

## 本期范围
- 至少 3 类工具能力
- 工具执行状态展示
- 工具结果展示
- 高风险操作确认
- 工具失败反馈

## 建议覆盖的用户任务
- 搜索一个问题并总结结果
- 记录一条笔记或待办
- 获取设备或系统基础信息

## 非目标
- 多端设备联动
- 插件市场
- 定时自动执行

## 验收标准
- 至少存在 3 个用户可见工具能力
- 用户发出任务型指令后，AI 可以实际调用工具并返回最终结果
- 用户能在界面上看到完整的工具执行过程
- 高风险工具在执行前必须确认
- 工具失败不会导致整个会话失控

## 测试方式
1. 输入需要搜索的信息任务，验证工具被调用并返回结果
2. 输入记录类任务，验证结果可见且可确认完成
3. 触发高风险工具，验证出现确认弹窗
4. 模拟工具失败，验证界面可见失败状态与重试入口
5. 连续触发多个工具任务，验证会话仍可继续
